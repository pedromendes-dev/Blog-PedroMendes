Este é o primeiro post do blog. A ideia é simples: registrar o que aprendo
construindo software e abrir as decisões técnicas que normalmente ficam só na
cabeça ou no Slack do trabalho.

## Sobre o que vou escrever

Os temas giram em torno do que faço no dia a dia:

- **Arquitetura**: DDD, Clean Architecture, quando aplicar e quando é overkill.
- **Backend**: .NET, Node.js, modelagem de dados, SQL Server.
- **Frontend**: React, TypeScript, micro frontends.
- **Infra**: Docker, Linux, CI/CD.

## Como o blog foi feito

Ele roda em [Astro](https://astro.build), gera HTML estático e é hospedado no
GitHub Pages. Cada post é um arquivo Markdown — escrever é só criar um `.md`
nesta pasta.

> Conteúdo bom é aquele que você gostaria de ter encontrado quando travou num
> problema. É esse o padrão aqui.

Bora começar.
